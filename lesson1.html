車位比喻篇：掌握「值」與「地址」的運作

🚗 什麼是「車位比喻」？

想像 記憶體 是一個巨大的停車場。每個 變數 就是一個車位。 基本型別像機車直接停入車位；物件像大巴士，車位放不下，所以只放 「取車單/門牌」 指向後方的 VIP 大車庫 (Heap)。

Java 原始碼模擬

char key = 'A'; // 小型機車
float pi = 3.14f; // 電動滑板車
String s = "Hi"; // 貨物箱 (取車單)
Person p = new Person(); // 豪華巴士 (取車單)
int[] row = {1, 2}; // 連號預訂位
String[] tags = {"A"}; // 裝滿取車單的連號櫃

類別與物件

類別是「設計圖」，物件是「實體成果」。

設計圖 (Class)
定義：有幾扇窗、什麼引擎、顏色屬性。
實體車 (Object)
真實存在於 Heap，可以發動、行駛。

程式進入點 (Main)

public static void main(String[] args) { ... }

為什麼要 Static？

因為程式剛啟動時，還沒 new 出任何東西。Static 讓 JVM 不需要「先買車」就能「進車庫」。

String[] args 是什麼？

這是啟動時餵給程式的「外部指令單」，也是一種裝滿字串地址的連號櫃。
